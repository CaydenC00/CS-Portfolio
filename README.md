# CS Program Portfolio - Sprint Review and Retrospective

## 1. Interpreting User Needs and Implementing Them
I interpret user needs by breaking down high-level requirements into clear, user-focused outcomes rather than relying on massive upfront documentation. In the SNHU Travel pilot, creating "user stories" (like filtering flight results by price) kept the focus entirely on the traveler's perspective. User stories help establish clear acceptance criteria early, allowing developers and testers to collaborate on the exact same day to ensure the implemented features match user expectations.

## 2. Program Development Approach
I approach program development using an iterative framework where the Software Development Life Cycle (SDLC) is compressed into short, manageable cycles. Instead of executing requirements, design, coding, and testing in long, separate phases over months, I implement a continuous lifecycle. This ensures that software increments are constantly delivered and tested, letting us catch technical bugs early and adapt to sudden changes—such as when we successfully absorbed GDPR data privacy compliance mid-development without stopping project momentum.

## 3. Future Agile Processes
In my future development work, I hope to fully incorporate digital visualization tools like **Jira Kanban boards** and key Scrum events including **15-minute Daily Scrums** and **Sprint Planning**. Using a Kanban board helps visually track progress from "To Do" to "Done," making bottlenecks easy to spot. Additionally, leveraging historical sprint velocity metrics ensures team commitments remain realistic and achievable.

## 4. Being a Good Software Team Member
Being a good team member in software development means removing departmental communication silos, focusing on collaboration rather than micromanagement, and maintaining supportive communication. Examples include proactively syncing with peers to clear technical blockers right after daily standups and building team confidence by encouraging peers to demo their working software directly to stakeholders to keep morale high.
